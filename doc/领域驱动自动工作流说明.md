# 领域驱动自动工作流说明

## 1. 目标

本轮新增能力面向这样的目标：

用户只输入一个领域或方向，例如：

- 龙虾餐饮
- 情感成长
- 门店运营
- 母婴护理

系统自动完成：

1. 扩展领域查询词
2. 搜索该领域的热点内容
3. 选出热度最高的几个热点
4. 提取关键词与钩子要素
5. 生成脚本主题与视频提示词包
6. 可选自动审批脚本
7. 可选自动生成视频
8. 记录本次工作流运行结果

## 3. 新接口

```text
POST /api/workflows/domain-auto-run
GET /api/workflows/runs
```

## 4. 请求示例

```json
{
  "domain": "龙虾门店运营",
  "platform": "douyin",
  "hotspot_count": 12,
  "top_n": 3,
  "content_type": "knowledge",
  "style": "clean",
  "video_style": "realistic",
  "duration": 30,
  "audience": "餐饮老板",
  "publish_goal": "涨粉",
  "auto_approve_script": false,
  "auto_generate_video": false
}
```

## 5. 返回内容

当前返回重点包括：

- 工作流运行记录 ID
- 扩展后的领域查询词
- 入选热点列表
- 关键词包
- 钩子关键词
- 视觉关键词
- 标题候选
- 脚本主题
- 视频提示词摘要
- 生成出的脚本状态
- 可选视频任务结果

## 6. 当前实现策略

### 6.1 热点阶段

当前会把领域词扩展成多组查询，例如：

- 原始领域词
- 领域 + 爆款
- 领域 + 热门
- 领域 + 选题
- 领域 + 用户画像
- 领域 + 发布目标

### 6.2 热点 provider 适配层

热点采集层已经抽成 provider 适配模式，当前支持：

- `douyin`
- `xiaohongshu`
- `bilibili`

目前默认还是 mock provider，但后续接真实采集器时，不需要重写业务层和工作流层。

### 6.3 排序阶段

当前热度分数采用简单加权：

- 播放量
- 点赞量
- 评论量
- 分享量

### 6.4 提示词阶段

当前提示词包主要从以下来源提取：

- 热点标题
- 热点 tags
- 热点分类
- 分析报告中的框架摘要
- reusable elements
- 风险提示

### 6.5 运行记录阶段

每次工作流执行都会落库到 `workflow_runs`，记录：

- 输入领域
- 目标平台
- 扩展查询词
- 选中的热点 ID
- 提示词包快照
- 生成出的分析 ID、脚本 ID、视频任务 ID
- 最终结果
- 错误信息

## 7. 当前与最终目标的差距

虽然已经具备“领域驱动工作流”骨架，但目前仍有几个地方属于可继续升级状态：

1. 热点采集仍是占位数据，不是真实平台采集
2. 关键词提取目前是规则化提取，不是更强的语义级抽取
3. 视频提示词包已经形成，但还没有做多版本 prompt 优选
4. 还没有加入发布效果反馈反哺下一轮选题
5. provider 适配层已建立，但真实平台抓取还未接入
6. 工作流运行记录已入库，但还没有做可视化工作台

## 8. 最适合继续优化的方向

建议下一步按这个顺序继续：

1. 接入真实平台热点采集器
2. 增加领域画像配置表
3. 增加关键词 / 钩子 / 镜头模板库
4. 增加一键批量跑多个领域
5. 增加“数据反馈 -> 下轮选题优化”闭环
6. 增加工作流运行历史管理页
