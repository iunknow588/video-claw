# 项目实现分析与操作说明

## 1. 结论先说

对照 `doc` 中的开发计划和检查项来看，当前 `src` 已经达到“可演示、可联调、可测试”的 MVP 后端原型水平，但还没有达到完整交付或正式生产可用状态。

更准确的判断是：

- 已具备主流程闭环
- 已补齐审核记录和成本记录
- 已补齐视频结果存储链路
- 已增加基础测试
- 仍缺真实 AI 平台接入、任务系统落地、权限和发布模块

## 2. 已实现能力

### 2.1 基础后端结构

当前已具备：

- FastAPI 主应用
- SQLAlchemy 异步数据库访问
- MySQL / Redis / Docker Compose 配置
- Alembic 迁移与启动前数据库连通性检查
- 日志与基础配置管理

### 2.2 核心数据模型

当前主数据线已经覆盖：

- `hotspot_items`
- `analysis_reports`
- `scripts`
- `video_tasks`
- `review_records`
- `cost_records`

这意味着热点、分析、脚本、视频、审核、成本六条主数据线已经建起来了。

### 2.3 核心流程链路

当前主流程已经具备：

1. 热点创建、查询、搜索、占位采集
2. AI 分析生成
3. 脚本生成
4. 脚本审核
5. 视频任务创建
6. 视频结果存储
7. 视频审核
8. 运营统计、审核记录、成本记录查询

## 3. 本轮完善与优化内容

### 3.1 修复代码可运行性

已修复早期源码中的转义和语法问题，当前代码已可编译。

### 3.2 补齐审核和成本留痕

已新增：

- 脚本审核记录入库
- 视频审核记录入库
- 分析、脚本、视频三个阶段的成本记录

### 3.3 补齐视频存储链路

原先视频结果只写占位 URL，现在已经改成：

1. 获取上游视频结果地址
2. 转为可落地的字节内容
3. 通过可配置存储后端写入
4. 将最终可访问 URL 写入 `video_tasks.video_url`

当前支持：

- `local`
- `github_release`
- `ipfs`
- `s3_compatible`

### 3.4 增加存储状态查询

已新增接口：

- `GET /api/operations/storage`

用于查看当前视频存储后端和脱敏后的配置状态，便于联调和排查。

### 3.5 增加测试

已增加测试覆盖：

- 热点搜索与占位采集
- 分析 -> 脚本 -> 视频主流程
- 审核记录与成本记录
- 本地视频存储
- S3 兼容存储的工厂选择与配置描述
- API 层主流程测试

### 3.6 接入真实 AI 接口骨架

当前已经补上：

- DeepSeek 客户端骨架
- GLM 客户端骨架
- Seedance 客户端骨架
- 超时、基础重试、错误回退

默认行为：

- 如果未配置 API Key，则自动回退到占位结果
- 这样可以继续本地联调，不会因为第三方平台未开通而卡死

### 3.7 增加 Alembic 初始化

当前已经补上：

- `alembic.ini`
- `src/alembic/env.py`
- 首版迁移脚本

并处理了异步数据库 URL 到 Alembic 同步迁移 URL 的兼容转换。

## 6. 操作说明

### 6.1 环境准备

进入目录：

```powershell
cd E:\2026OPC大赛\龙虾流程
```

创建虚拟环境并安装依赖：

```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install -r src/requirements.txt
```

### 6.2 启动基础服务

```powershell
docker-compose up -d db redis
```

### 6.3 配置 `.env`

最小示例：

```env
DEBUG=true
DATABASE_URL=mysql+aiomysql://root:YOUR_PASSWORD@127.0.0.1:3306/ai_video_prod?charset=utf8mb4
DEEPSEEK_API_KEY=
GLM_API_KEY=
SEEDANCE_API_KEY=
AI_USE_PLACEHOLDER_WHEN_UNCONFIGURED=true

VIDEO_STORAGE_BACKEND=local
MEDIA_ROOT=runtime/media
MEDIA_URL_PREFIX=/media
MEDIA_BASE_URL=
```

### 6.4 启动应用

```powershell
python -m uvicorn main:app --app-dir src --reload
```

### 6.5 打开接口文档

访问：

```text
http://127.0.0.1:8000/docs
```

### 6.6 推荐操作顺序

1. 调用 `POST /api/hotspots/fetch`
2. 用返回的 `hotspot uuid` 调用 `POST /api/analysis`
3. 用返回的 `analysis uuid` 调用 `POST /api/scripts`
4. 调用 `POST /api/scripts/review/{script_id}`
5. 调用 `POST /api/videos`
6. 调用 `POST /api/videos/review/{task_id}`
7. 调用 `GET /api/operations/summary`
8. 调用 `GET /api/operations/reviews`
9. 调用 `GET /api/operations/costs`
10. 调用 `GET /api/operations/storage`

### 6.7 视频存储选择建议

当前最推荐：

1. 开发与比赛阶段：`local`
2. 正式部署阶段：`s3_compatible`

谨慎使用：

- `github_release`：适合演示资产，不适合高频视频写入
- `ipfs`：适合展示或实验，不适合作为当前主存储

### 6.8 数据库迁移

执行：

```powershell
python -m alembic -c alembic.ini upgrade head
```

如需新增迁移：

```powershell
python -m alembic -c alembic.ini revision -m "your change"
```

### 6.9 运行测试

```powershell
python -m pytest -c pytest.ini src/tests -q
```
