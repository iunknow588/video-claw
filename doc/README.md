# 文档总览

`doc/` 目录分为三类内容：

1. 当前可执行文档：描述现在这套系统如何启动、如何联调、目录实际长什么样。
2. 设计与重构文档：位于 `doc/design/`，用于说明当前生效的架构边界、模块归属和整改规则。
3. 历史草案与参考资料：保留讨论过程和旧方案，不作为当前实现依据。
