### 0.3 优化建议

1. 每个 LEAD 只接收上游 bundle，只输出自己的 bundle，不直接依赖上上游 model。
2. 各 skill 统一补齐 `input_schema` / `output_schema` / `status` / `error` / `trace_id`。
3. CEO 只做 plan 与 dispatch，不直接做热点、分析、脚本、视频、发布的具体业务。
4. LOG 以 step 级事件为主，run 级汇总为辅，同时支持回放、重试和验收。
5. 先补 `发布 LEAD` 和 `step 日志`，因为它们决定了闭环和验收是否成立。
6. 再把 `workflow.py` 拆成 `CEO skill` + 各 LEAD orchestrator + leaf executor。
7. 最后补真实平台 adapter 与 mock 的切换层。

### 0.4 LEAD 级叶子任务再拆分建议

| LEAD | 建议继续拆的叶子任务 | 备注 |
|------|---------------------|------|
| 调研 LEAD | `DomainQueryExpansionSkill` / `HotspotCollectionSkill` / `HotspotDedupSkill` / `HotspotRankingSkill` / `HotspotSnapshotSkill` | 让采集、去重、排序、落库分开 |
| 分析 LEAD | `HotspotStructureSkill` / `HookExtractionSkill` / `EmotionCurveSkill` / `RiskExtractionSkill` / `ReusableElementSkill` / `AnalysisPersistSkill` | 分析结果要结构化，便于下游消费 |
| 研发 LEAD | `PromptPackageSkill` / `TitleCandidateSkill` / `PromptValidationSkill` / `PromptVersionSkill` | 保证 prompt 可复用、可回滚 |
| 生产 LEAD | `ScriptDraftSkill` / `ScriptReviewSkill` / `VideoTaskSkill` / `VideoProcessSkill` / `VideoReviewSkill` / `AssetStorageSkill` / `RetryRecoverySkill` | 把创作、审核、生成、存储、重试拆开 |
| 发布 LEAD | `PublishPlanSkill` / `PlatformAdapterSkill` / `PublishExecuteSkill` / `PublishCallbackSkill` / `PublishHistorySkill` / `RetryRecoverySkill` | 这是当前最需要补齐的闭环 |
| LOG skill | `RunEventSkill` / `StepEventSkill` / `CostEventSkill` / `ErrorEventSkill` / `ReplaySkill` / `QuerySkill` | 统一日志、审计、回放、查询 |
| CEO skill | `PlanSkill` / `DispatchSkill` / `DependencySkill` / `ResumeSkill` / `FailoverSkill` / `ResultMergeSkill` | 只做全局调度，不碰具体业务细节 |

### 0.5 推荐实施顺序

1. 先补 `发布 LEAD`。
2. 再补 `LOG step/event` 级记录。
3. 然后拆 `workflow.py`，让 CEO 只保留调度。
4. 接着把各 LEAD 的 bundle 输入输出定死。
5. 最后把真实平台 adapter、重试、回放、验收测试补齐。
