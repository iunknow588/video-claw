# 代码重构落地清单

## 1. 迁移目标

把现有散落在 `services`、`skills`、`models`、`leaders`、`api` 等位置的功能，统一并入八个顶层业务模块：

- `app/CEO`
- `app/CMO`
- `app/CIO`
- `app/CTO`
- `app/CFO`
- `app/CHO`
- `app/CAO`
- `app/CQO`

## 2. 迁移原则

1. 先定归属，再搬代码。
2. 一个能力只保留一个主入口。
3. 入口文件只负责导出，不负责堆业务。
4. 模块内再拆子目录，不再扩展平级体系。

## 3. 迁移映射

### 3.1 业务技能

- `app/skills/lead/research*` -> `app/CTO` 或 `app/CIO` 归口整理
- `app/skills/lead/analysis*` -> `app/CTO` / `app/CQO` / `app/CIO`
- `app/skills/lead/production*` -> `app/CAO`
- `app/skills/lead/publish*` -> `app/CAO`
- `app/skills/lead/qa*` -> `app/CQO`
- `app/skills/lead/cfo*` -> `app/CFO`
- `app/skills/lead/cio*` -> `app/CIO`
- `app/skills/lead/cho*` -> `app/CHO`
- `app/skills/ceo*` -> `app/CEO`
- `app/skills/log*` -> `app/CIO`

### 3.2 领导与组织

- `app/leaders` 相关能力 -> `app/CEO`
- 部门治理与组织配置 -> `app/CEO`
- 部门领导对象 -> 对应部门目录内部

### 3.3 服务层

- `app/services/orchestration*` -> `app/CEO`
- `app/services/workflow*` -> `app/CEO` / `app/CIO`
- `app/services/script*` -> `app/CAO` / `app/CTO`
- `app/services/video*` -> `app/CAO`
- `app/services/storage*` -> `app/CIO` / `app/CAO`
- `app/services/audit*` -> `app/CIO`
- `app/services/publish*` -> `app/CAO`
- `app/services/analysis*` -> `app/CTO`
- `app/services/finance*` -> `app/CFO`

### 3.4 数据模型

- `app/models/*` 依照业务归属并入对应模块内部
- 工作流、日志、资产、报告等模型优先归到 `app/CIO`
- 视频、脚本、分析、审核模型按业务链路归到对应模块

### 3.5 API

- `app/api` 只保留门面职责
- 真正的业务接口实现归入对应模块内部
- 控制类接口归 `app/CEO`
- 展示类接口归 `app/CMO`
- 交付类接口归 `app/CAO`
- 质量类接口归 `app/CQO`

## 4. 清理顺序

1. 先建立目标目录结构。
2. 再逐个迁移业务实现。
3. 接着修正 import 路径。
4. 然后删掉旧的平级业务体系。
5. 最后清理重复 `__init__.py` 和无效导出。

## 5. 代码验收点

- 不再新增 `app/skills/lead/...` 这类旁路业务目录
- 不再新增平级大写业务目录之外的独立业务体系
- 每个新文件都能一眼看出归属模块
- `main.py` 只负责启动，不承载业务逻辑
