# 运行期资产目录说明

## 目标

把源码、配置、设计文档与运行期生成资产彻底分开，避免媒体文件、日志文件和设计草稿混在一起。

## 统一目录

运行期资产统一收敛到仓库根目录的 `runtime/`：

```text
runtime/
├── logs/
└── media/
    ├── audio/
    ├── subtitles/
    ├── renders/
    └── videos/
```

## 实际作用

### `runtime/logs/`

- 应用运行日志目录
- 默认日志文件：`runtime/logs/app.log`

### `runtime/media/audio/`

- 语音旁白输出
- 典型文件：`trace-voice-1.wav`

### `runtime/media/subtitles/`

- 字幕生成输出
- 典型文件：`trace-subtitle-1.srt`

### `runtime/media/renders/`

- 渲染清单与中间描述文件
- 典型文件：`trace-render-1.json`

### `runtime/media/videos/`

- 本地视频成品、预览视频、占位视频
- 通过 `/media/videos/...` 对外暴露

## 责任归属

- `CIO/services/runtime_assets/`：统一解析项目根目录下的运行期路径
- `CIO/services/storage/`：管理媒体资产根目录与公开 URL
- `CEO/core/logging/`：管理日志落盘位置
- `CEO/app.py`：挂载 `/media` 静态资源入口

## 规则

1. `src/` 只放源码、配置、测试和启动入口
2. `doc/design/` 只放设计文档
3. `runtime/` 只放运行期资产
4. 运行期生成文件不入库，只保留目录骨架

## 同步调整
