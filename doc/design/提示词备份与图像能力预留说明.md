# 提示词备份与图像能力预留说明

## 1. 目标

当前系统已经打通文本主链路，但 `prompt_package` 仍然只有单份主提示词。

为了降低单次生成不稳定、后续模型切换成本高、以及图片能力无法挂接的问题，`CTO` 规划层需要同时产出：

- 主脚本主题
- 备用脚本主题
- 主视频提示词
- 备用视频提示词
- 备用图片提示词

## 2. 结构约定

`prompt_package` 至少包含以下字段：

- `script_topic`
- `script_topic_variants`
- `video_prompt`
- `video_prompt_variants`
- `image_prompt_variants`

其中：

- `script_topic` 是当前默认执行主题
- `script_topic_variants` 供脚本改写、QA 返工、人工切换时复用
- `video_prompt` 是当前默认视频提示词
- `video_prompt_variants` 供未来多视频模型或多镜头策略复用
- `image_prompt_variants` 供封面图、分镜图、关键视觉图生成复用

## 3. 部门职责

- `CSO/trend_intelligence`
  - 负责根据热点与分析结果生成提示词基底
- `CTO/use_cases/prompt_planning`
  - 负责把单份提示词组织成正式的规划包
- `COO`
  - 当前只消费主脚本主题
  - 后续可按需要切换到备用主题或图片提示词

## 4. Qwen-Image 的系统定位

`Qwen-Image-2512` 在当前结构中更适合承接：

- 封面图
- 关键视觉图
- 分镜预览图
- 海报图

不应直接承担：

- 完整视频生成
- 成片时序建模
- 镜头连续运动生成

因此系统上应保持：

1. 图片模型归入 `COO/asset_management` 或未来的图像资产服务
2. 视频模型仍归入 `COO/video_production`
3. `prompt_package` 同时给出视频 prompt 与图片 prompt，避免未来再回头重构提示词结构
