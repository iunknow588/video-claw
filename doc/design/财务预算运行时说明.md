# 财务预算运行时说明

## 目标

在当前 `CFO` 部门结构下，把预算与阈值策略从散落的全局 `settings` 读取中收拢，形成统一的财务运行时入口。

## 问题

优化前，以下模块分别直接读取平铺配置：

1. `CFO/services/finance/`
2. `CFO/skills/verify_balance/`
3. `CIO/services/operations/`

这些模块都依赖：

- `DAILY_BUDGET`
- `COST_WARNING_THRESHOLD`
- `COST_ALERT_THRESHOLD`
- `COST_CRITICAL_THRESHOLD`

结果是：

1. 预算口径分散在多个部门。
2. 财务策略虽然属于 `CFO`，却没有统一的运行时表达。
3. 其他部门想引用预算规则时，只能回到全局平铺字段。

## 方案

在 `CFO` 下补充统一的财务运行时：

```text
departments/CFO/services/
  finance_runtime/
    __init__.py
```

该运行时负责：

1. 从 `CFO` 配置域读取预算与阈值。
2. 输出统一的 `FinanceRuntime` 快照。
3. 提供预算占用率与告警级别判断。

## 职责边界

### CFO

`CFO/services/finance_runtime/` 是财务策略唯一入口，负责：

1. 日预算
2. 预警阈值
3. 告警阈值
4. 危急阈值
5. 占用率与风险级别计算

### 业务服务

业务服务不再自己解释财务配置：

- `FinanceService` 只负责汇总支出与交易数据。
- `VerifyBalanceSkill` 只负责核验是否可放行。
- `OperationsService` 只负责展示运营指标。

## 结果

调整后形成清晰依赖关系：

1. `CFO` 拥有预算策略。
2. `CIO` 如需展示预算使用情况，依赖 `CFO` 预算运行时。
3. `skills` 与 `services` 都不直接拼预算阈值规则。

这使预算规则后续扩展更简单，例如：

1. 增加部门级预算。
2. 增加按 workflow 类型区分的预算池。
3. 增加不同告警级别下的自动管控动作。
